<p>Please, report any security issue to <code>opensourcegroup@netcracker.com</code> where the issue will be triaged appropriately.</p>
<p>If you know of a publicly disclosed security vulnerability please IMMEDIATELY email <code>opensourcegroup@netcracker.com</code><br>

<h1 class="toc-header" id="security-release-process"><a href="#security-release-process" class="toc-anchor">¶</a> Security Release Process</h1>
<p>If the vulnerability is found in the latest stable release, then it would be fixed in patch version for that release.<br>
E.g., issue is found in 2.5.0 release, then 2.5.1 version with a fix will be released.<br>
By default, older versions will not have security releases.</p>
<p>If the issue doesn't affect any existing public releases, the fix for medium and high issues is performed<br>
in a main branch before releasing a new version. For low priority issues the fix can be planned for future releases.</p>
